## Project quick-reference

- **Routes + folder structure**: `README.md`
- **Auth flow**: `src/lib/auth.ts`, `src/lib/session.ts`, `proxy.ts`
- **API client**: `src/lib/api.ts` (`apiGet`/`apiPost` against `NEXT_PUBLIC_API_URL`). `getEmail()` bootstraps by fetching `/api/me` (Next.js route that decodes the httpOnly session cookie) — wrapped in a 10s `AbortController` so a hung response (e.g. an aggressive privacy extension blocking the path) doesn't permanently stall every API call. Failed lookups are NOT cached (`lookupFailed` flag): the next call retries instead of being stuck on `null`. `apiGet` has its own 45s timeout that throws a typed `API timeout after Ns: <path>` error on abort. **Note: `apiPost` / `apiPut` / `apiPatch` / `apiDelete` don't yet share this timeout treatment** — if you add one, mirror the `apiGet` pattern.
- **Shared types**: `src/lib/types.ts` — mirror of `backend/app/schemas.py`. When you add or rename a field in either place, update both.
- **Store pattern (CP v2)**: `src/app/(app)/capacity-planning/_store.tsx` is a React context backed by `localStorage`. Every CP v2 page reads + writes through this store. When the backend ships `/api/cp2/*` endpoints, swap `localStorage` calls for `apiGet`/`apiPost`; UI stays the same.
- **Filter bar**: `src/components/dashboard/FilterBar.tsx` fans out filtered client list + date range via `onFilterChange` + `onDateRangeChange`. Dashboards that want time-aware charts must consume both. Status options: `All` / `Active` / `Soon to be active` (`SOON_TO_BE_ACTIVE`) / `Inactive/Completed`.

- **Triage variance math (single source of truth)**: every "is this client off target?" calculation on /overview and the per-client cards uses **cumulative variance through end of the current contract Q** — `Σ delivered actuals + Σ remaining-month projections IN this Q − Σ contracted invoicing from start through end of this Q`. Catch-up over-delivery in one Q nets against earlier-Q deficits so a `−14 last Q + 14 this Q` reads as 0, mirroring the spreadsheet's Variance row byte-for-byte. **Tier buckets are SYMMETRIC and magnitude-based** (as of 0.3.21 — being far ahead is flagged like being far behind, since over-delivered work isn't billed yet): the canonical classifier is `varianceTier(variance, isNew)` in `shared-helpers.tsx` → `v = 0` On track (green) · `1 ≤ |v| ≤ 5` Within limit (amber) · `v > 5` Ahead (red) · `v < −5` Behind (red) · 1st-Q clients excepted to a blue "1st Q" (never red). EVERY variance chip / label / cell color routes through `varianceTier` / `varianceTierColor` / `varianceTierBg` — don't re-inline the buckets. `isOffTarget(v, isNew)` (`|v| > 5 && !isNew`) drives the Needs Attention + Pod Attention selection. Implementations: `computeCurrentQ()` + `clientCurrentQTier()` in `DeliveryOverviewCards.tsx` (Triage cards; note `clientCurrentQTier`'s historical `"behind"` key now means off-target in EITHER direction) and `quarterMetaFromPeriods()` in `ClientDeliveryCards.tsx` (per-client cards). The Monthly detail popover's "Variance" column also uses cumulative math — don't switch any of these surfaces to per-Q in isolation without updating the others.
- **"New (1st Q)" escape hatch**: clients whose current contract Q is their first (Q1 of year 1) always look behind on contracted invoicing because they haven't had time to ramp. Triage cards + `ClientDeliveryCard` classify them as a 4th tier (`new`, blue chip color `#8FB5D9`) instead of lumping them into Behind. - **`FilterBar` zero-match fallback**: when non-date filters yield no clients, the date range resets to "All Time" so the user isn't stuck with an empty window. Mirror this if you add new filter dimensions.
- **Analytics tracking** (`src/lib/analyticsClient.ts`): `trackEvent("EventName", { route, section_id?, props? })` queues an event and flushes on a 10s timer OR at 5 events. `useAnalytics()` is the hook wrapper. Section dwell tracking lives in `src/lib/useSectionDwell.ts` — `useSectionDwell(sectionId, ref)` and `useSectionDwellById(sectionId)` emit `SectionEntered` on first visibility (IntersectionObserver, 50% threshold) and `SectionViewed` with `dwell_ms` on exit, paused on `visibilitychange`. A `PageViewTracker` client component is mounted in `(app)/layout.tsx` and fires `PageView` on every route change (Suspense-wrapped — uses `useSearchParams`); it also installs the Recharts dev-warning filter (`src/lib/suppressRechartsWarning.ts`) once on boot. Inline `trackEvent` calls live at the choke points: `FilterBar.updateParams` → FilterChanged, `SyncControls.handleSyncClick` → SyncClicked, `PeriodSnapshotSection.openCell` → DrillDownOpened (variant=client/goals/lastQ/currentQ/lifetime), `useOverviewComments.create/update/resolve/remove` → CommentPosted / CommentEdited / CommentResolved / CommentDeleted, `ProductionTrendChart.ViewModeToggle` → ClickInteraction (label=production-history.view-toggle.\*). The `/admin/analytics` page has two tabs (Dashboard / Tracking Coverage), is gated on the `admin.analytics` view (forced-revoked on every non-admin group in `access.py`), and consumes `GET /api/analytics/summary?range=…&groups=<csv>` — the GroupFilter component reads `/api/access/groups` and lets admins filter every rollup by RBAC group membership. All chart tooltips on this page render via a shared `DarkTooltip` content component so series colours don't bleed into label/value text. - **Tooltip body**: import `TooltipBody` from `shared-helpers.tsx` for every metric tooltip — uppercase mono title + 2–3 short bullets. Trigger should be tight (no large hit-area padding). **`DataSourceBadge` is now a no-op** (returns null) so existing call sites compile but render nothing — the LIVE / source chip was deemed visual noise across the Hub.
- **Tooltip copy style** (applies to every dashboard surface — Overview, D1, D2, shared cards, charts, Admin pills). Audience is non-technical operators. Strict rules:
  1. Max **3 bullets** per tooltip. Title ≤ 4 words, sentence case.
  2. Each bullet ≤ **10 words**. One short declarative thought, terminal period.
  3. **No code, table, or sheet-tab names** in user-facing strings.
  4. Math is fine, plain symbols: `delivered ÷ invoiced`, `Σ`, `±5`. Don't write out "variance" jargon — say "behind / ahead".
  5. End with an action when the row is clickable: `Click a row to jump.`
  6. CP2 sub-pages are an explicit exception while they're [Proposal] — their tooltips can keep schema references (`cp2_fact_*`) until that subsystem ships to end users.
