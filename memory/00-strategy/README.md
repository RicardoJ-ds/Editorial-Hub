# 00 · Strategy — load first

The current playbook. What someone needs to understand *the plan* before doing anything.

Put here: the framework/architecture design, business logic & domain context, the analysis
or build strategy, and open questions (`open_q_<topic>.md`). Update these **in place** as the
strategy evolves — `log.md` carries the history.

Suggested starter files (rename/replace freely):
- `framework_design.md` — the master plan and how the pieces fit together.
- `business_logic_<area>.md` — domain rules, biases, targets, definitions.
- `open_q_<topic>.md` — unresolved questions (promote to a `20-decisions/` file when answered).
