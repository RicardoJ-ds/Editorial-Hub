# 10 · Reference — stable look-ups

Things you look up but don't re-litigate. Lower-churn than strategy.

Put here: the project overview (scope, hard rules, status), data/source maps (lineage +
trust ratings), glossaries, and schema/column dictionaries.

Suggested starter files:
- `project_overview.md` — scope, hard rules ("do NOT…"), data sources, current status.
- `data_source_map.md` — where data comes from, its lineage, and how trustworthy each source is.
- `glossary.md` — canonical terms so naming stays consistent.
